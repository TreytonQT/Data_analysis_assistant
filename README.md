# 开发员销售数据看板

本项目是一个本地 Streamlit 看板，用于手动上传多个月的“业绩报表”CSV，并通过配置表公式生成月度、开发员、店铺、开发员+店铺维度分析。

## 启动

```powershell
python -m pip install -r requirements.txt
streamlit run app.py
```

## 使用方式

1. 左侧导航切换 `首页`、`上传中心`、`配置中心`。
2. 在 `上传中心` 上传一个或多个业绩报表 CSV；报表会保存到 `data/reports/`，刷新或重新打开看板后仍可继续分析。
3. 同一月份重复上传会替换旧记录，只保留最新版。
4. 在 `配置中心` 可选上传指标公式配置；不上传时使用 `configs/metrics_config.csv`。
5. 店铺配置、目标配置和提成配置在 `配置中心` 直接维护，保存后写入 `configs/`。
6. 在 `首页` 顶部筛选栏选择月份、开发员、店铺所属部门、店铺类型，查看总览、趋势、开发员、店铺、开发员+店铺和异常预警。

## 指标公式配置

文件：`configs/metrics_config.csv`

必填列：

- `指标名称`
- `显示分组`
- `公式`
- `格式`
- `排序`
- `是否启用`

公式示例：

```text
range_sum("销售额--FBA销售额", "COD")
safe_divide(sum([毛利润]), range_sum("销售额--FBA销售额", "COD"))
if(safe_divide(sum([毛利润]), sum([销售额--FBA销售额])) < 0.15, 1, 0)
```

支持能力：

- 字段引用：`[字段名]`
- 四则运算、括号、比较、布尔运算
- 函数：`sum()`、`mean()`、`min()`、`max()`、`count()`、`nunique()`、`abs()`、`round()`、`safe_divide()`、`if()`、`range_sum()`
- 字符串比较：`[店铺类型] == "本土"`

安全限制：

- 不执行任意 Python 代码。
- 只能使用白名单函数和 CSV/config 中存在的字段。
- 缺失字段会在页面明确提示。

## 业务配置

店铺配置在网页中维护，字段为：`店铺名`、`店铺类型`、`停提款时间`、`店铺所属部门`。

`停提款时间` 为空表示该店一直计入常规看板；填写月份后，从该月份开始包含当月的数据不计入首页常规看板，仅在提成预估中单独列出停提款店铺缺提成。

目标配置在网页中维护，字段为：`开发员`、`目标业绩`、`目标毛利率`。

部门费用率配置在网页中维护，字段为：`月份`、`部门`、`费用率`。提成计算会按店铺所属部门匹配对应月份的费用率。

提成配置在网页中维护，字段为：`月份`、`开发员`、`库存计提`、`弃置`、`职位提点`。

配置未命中时不会阻断基础分析，相关字段显示为“未配置”或空值。
